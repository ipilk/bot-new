# بوت ديسكورد لتشغيل الصوت من يوتيوب 🎵

بوت ديسكورد بسيط يقوم بتشغيل الصوت من روابط يوتيوب في القنوات الصوتية باستخدام Slash Commands.

## المتطلبات الأساسية 📋

- Python 3.8 أو أحدث
- FFmpeg
- توكن بوت ديسكورد
- حساب على Railway.app (للنشر)

## التثبيت المحلي 🚀

1. قم بتثبيت المتطلبات:
```bash
pip install -r requirements.txt
```

2. قم بإنشاء ملف `.env` وأضف توكن البوت الخاص بك:
```
DISCORD_TOKEN=your_discord_bot_token_here
```

## الأوامر المتاحة 🎮

- `/play [رابط_يوتيوب]` - تشغيل الصوت من رابط يوتيوب
- `/stop` - إيقاف التشغيل والخروج من القناة الصوتية

## النشر على Railway 🚂

1. قم بإنشاء حساب على [Railway](https://railway.app)

2. قم بإنشاء مشروع جديد على Railway:
   - اختر "Deploy from GitHub repo"
   - اختر المستودع الخاص بالبوت
   - انتظر حتى يتم استيراد المشروع

3. أضف المتغيرات البيئية:
   - انتقل إلى Variables في لوحة التحكم
   - أضف `DISCORD_TOKEN` مع القيمة الخاصة بتوكن البوت

4. تأكد من تفعيل الـ Slash Commands:
   - قم بزيارة [Discord Developer Portal](https://discord.com/developers/applications)
   - اختر تطبيقك
   - في قسم "Bot"، فعّل جميع "Privileged Gateway Intents"
   - في قسم "OAuth2 > URL Generator":
     - اختر `applications.commands` و `bot`
     - اختر الصلاحيات المطلوبة للبوت
     - استخدم الرابط الناتج لإضافة البوت إلى سيرفرك

5. انتظر حتى يتم نشر البوت (قد يستغرق بضع دقائق)

## ملاحظات مهمة ⚠️

- تأكد من أن البوت لديه الصلاحيات المناسبة في سيرفر الديسكورد
- يجب أن تكون متصلاً بقناة صوتية قبل استخدام أمر التشغيل
- قد يستغرق ظهور الـ Slash Commands حتى ساعة بعد إضافة البوت للمرة الأولى
- في حال واجهت مشاكل في التشغيل على Railway:
  - تأكد من أن جميع الملفات (`railway.toml`, `Procfile`, `requirements.txt`) موجودة في المستودع
  - تأكد من أن المتغيرات البيئية تم إضافتها بشكل صحيح
  - راجع سجلات التشغيل (Logs) في لوحة تحكم Railway للتعرف على أي أخطاء
